# 4. 시연 시나리오

## 시연 목적
- CCTV(AI) + 임베디드(ROS2/MQTT) + 백엔드 + 관리자웹 + 모바일앱이 연동되어
  입차/주차/결제/출차 흐름이 정상 동작함을 검증한다.

## 0) 사전 준비
1. 백엔드 실행: `be/S14P11A201/zerojuice`
2. 관리자 웹 실행: `fe/S14P11A201/my-react-app`
3. 모바일 앱 실행: `app/S14P11A201/smart-parking`
4. 임베디드 노드 실행: `emb/...` (RF2O/Nav2/MQTT publish)
5. AI 파이프라인 실행: `ai/S14P11A201/parking_slot_detection.py`

## 1) 관리자 웹 시연

### STEP-W1 로그인
- 경로: `/`
- 액션: 계정 입력 후 로그인 버튼 클릭
- 기대결과: 대시보드(`/dashboard`) 이동

### STEP-W2 대시보드 모니터링
- 경로: `/dashboard`
- 액션: 실시간 슬롯/요약/입출차 현황 확인
- 기대결과: AI/백엔드/SSE 갱신 데이터 표시

### STEP-W3 CCTV 화면 확인
- 경로: `/cctv`
- 액션: CCTV 프레임과 슬롯 상태 변경 확인
- 기대결과: `video_frame` 및 슬롯 상태 반영

### STEP-W4 트래픽 로그 확인
- 경로: `/trafficlog`
- 액션: 입출차 이벤트 로그 확인
- 기대결과: RFID 및 주차 이력 이벤트 누적

### STEP-W5 설정 변경
- 경로: `/setting`
- 액션: 요금/시간/CCTV URL 등 설정 변경 후 저장
- 기대결과: 설정 API 반영 및 후속 계산에 반영

## 2) 모바일 앱 시연

### STEP-M1 로그인
- 화면: `LoginScreen`
- 액션: 휴대폰 로그인 또는 카카오 로그인
- 기대결과: `MainDashboard` 진입

### STEP-M2 입차 요청
- 화면: `MainDashboard`
- 액션: 입차 시작 버튼 클릭 -> 확인 모달 승인
- 기대결과: `ParkingProgressScreen` 이동, 실시간 위치 갱신(SSE)

### STEP-M3 주차 완료 이벤트
- 화면: `ParkingProgressScreen`
- 액션: 차량이 목표 슬롯 도착
- 기대결과: `parking-complete` 이벤트 수신, 상태 `PARKED`

### STEP-M4 결제
- 화면: `PaymentScreen`
- 액션: 결제수단 선택(카카오/카드) 후 결제
- 기대결과: 결제 성공 콜백/딥링크 처리, 결제 완료 상태

### STEP-M5 출차
- 화면: `MainDashboard` 또는 진행 화면
- 액션: 출차 요청
- 기대결과: `exitlog` 이벤트 수신 후 세션 종료

## 3) 연동 검증 포인트
- AI -> Backend: `/api/v1/parking-slots/status`, `/api/v1/parking-summary`
- Emb -> MQTT -> Backend: `car/{carNo}/pose`, `car/{carNo}/command`, `rfid/enter`, `rfid/exit`
- Backend -> App/Web: `/api/v1/sse/**` 실시간 이벤트
- App/Web -> Backend: 인증/설정/주차/결제 API

## 4) 장애 대응 체크
- SSE 수신 안됨: Nginx proxy buffering/timeout 확인
- 주차상태 갱신 안됨: AI `THRESH`/ROI/카메라 인덱스 확인
- 차량위치 미표시: Emb `/amcl_pose`와 MQTT 토픽 확인
- 결제 실패: PortOne 키/PG 코드/딥링크 스킴(`zerojuice://`) 확인

## 5) 주차로직 검증 포인트

### 목적
입구, 출구에서 가까운 곳부터 우선적으로 주차 배정하는 로직을 검증

### 시나리오 1: B2, C1에 주차된 차량이 있는 경우

**상황**
- 기존 주차: B2, C1
- 입구: A 구역 근처, 출구: C 구역 근처

**시연 순서**
1. B2, C1 위치에 장애물 배치 
2. 모바일 앱에서 입차 요청
3. 백엔드가 남은 주차칸들 중 입/출구와 가장 가깝고 장애물이 없는 슬롯 자동 배정
4. 기대결과: 
   - B2, C1이 occupied로 표시됨
   - 새 차량은 입구/출구와의 거리를 고려한 최적 슬롯에 배정 (B1)
   - 주차 경로와 최종 슬롯 위치를 CCTV/관리자 웹에서 확인

### 시나리오 2: B1, B2, C1에 주차된 차량이 있는 경우

**상황**
- 기존 주차: B2, C1
- B1에 추가 주차가 되어서 B1, B2, C1 모두 장애물/occupied 상태

**시연 순서**
1. B1에 새로운 장애물 배치
2. 현재 상태: B1, B2, C1 모두 사용 불가 (occupied)
3. 모바일 앱에서 입차 요청
4. 백엔드 주차로직:
   - 입/출구에서 가장 가깝고 빈 슬롯 검색 → A1, C2 중 가장 가까운 곳
5. 기대결과:
   - B1, B2, C1이 모두 occupied/유사 상태로 표시
   - 새 차량은 출구 또는 입구와의 거리를 고려한 차선의 슬롯 배정 (C2)
   - 주차 배정 로직의 우선순위가 명확히 드러남
