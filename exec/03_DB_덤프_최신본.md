# 3. DB 덤프 파일 최신본

## 현재 저장소 상태
- `*.sql`, `*.dump`, `*.bak` 형식의 DB 덤프 파일은 저장소 내에서 확인되지 않음.
- 따라서 본 문서는 "최신 덤프 생성/복원 절차"와 "제출 위치"를 정의함.

## 1) 최신 덤프 생성 (MySQL)

```bash
mysqldump -h <DB_HOST> -P <DB_PORT> -u <DB_USER> -p \
  --databases zerojuice \
  --routines --triggers --events \
  --single-transaction --set-gtid-purged=OFF \
  > zerojuice_YYYYMMDD_HHMM.sql
```

권장
- 파일명 예: `zerojuice_20260208_2200.sql`
- 생성 후 압축: `zip zerojuice_20260208_2200.zip zerojuice_20260208_2200.sql`

## 2) 복원 절차

```bash
mysql -h <DB_HOST> -P <DB_PORT> -u <DB_USER> -p < zerojuice_YYYYMMDD_HHMM.sql
```

## 3) GitLab 제출 규칙(포팅 제출용)
- `exec` 폴더 하위에 덤프를 두지 말고, 용량 정책에 맞춰 별도 보관소/릴리즈 아티팩트로 관리 권장.
- 제출 문서에는 아래만 기재
  - 덤프 생성 시각
  - 생성 환경(DB host masked)
  - 스키마 버전 또는 커밋 해시

## 4) 확인 항목
- `parking_slots`, `parking_histories`, `cars`, `users`, `settings` 테이블 포함 여부 확인.
- 문자셋(`utf8mb4`)과 타임존(Asia/Seoul) 일치 여부 확인.
