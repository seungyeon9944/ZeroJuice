# 2. 프로젝트 외부 서비스 정보

## 1) 백엔드/인프라 연동

### MySQL
- 용도: 사용자/차량/주차상태/이력/설정 저장
- 참조 파일
  - `be/S14P11A201/zerojuice/src/main/resources/application.yml`
  - `be/S14P11A201/zerojuice/src/main/resources/application-local.yml`

필요 정보
- Host, Port, DB명
- DB 계정/비밀번호
- 권한: 앱 스키마 읽기/쓰기

### Redis
- 용도: 토큰(블랙리스트/리프레시) 관리
- 참조 파일
  - `be/S14P11A201/zerojuice/src/main/resources/application.properties`

필요 정보
- Host, Port, Password

### MQTT Broker
- 용도: 차량 제어 명령/차량 위치/RFID 이벤트 송수신
- 기본 토픽(코드 기준)
  - `car/%s/command`
  - `car/+/pose`
  - `car/{carNo}/pose`
  - `car/+/ack`
  - `rfid/exit`, `rfid/enter`
- 참조 파일
  - `be/S14P11A201/zerojuice/src/main/resources/application.yml`
  - `emb/S14P11A201/mqtt.py`
  - `emb/orin_car/mqtt_pose_publisher.py`

필요 정보
- Broker URL/IP, Port
- 인증 계정(ID/PW)
- 차량번호 기반 토픽 네이밍 규칙

### SSE (Server-Sent Events)
- 용도: 앱/웹 실시간 상태 스트리밍(주차상태, 차량위치, 주차완료, 출차로그, 요약)
- 주요 엔드포인트
  - `/api/v1/sse/parking-slots`
  - `/api/v1/sse/parking-histories`
  - `/api/v1/sse/parking-summary`
  - `/api/v1/sse/dashboard`
  - `/api/v1/sse/app/parking-status/{userId}`
  - `/api/v1/sse/app/parking-location/{userId}`
  - `/api/v1/sse/app/parking-complete/{userId}`
  - `/api/v1/sse/app/payment-flow/{userId}`
  - `/api/v1/sse/app/exitlog/{userId}`
- 참조 파일
  - `be/S14P11A201/zerojuice/src/main/java/com/zerojuice/domain/sse/controller/SseController.java`
  - `app/S14P11A201/smart-parking/src/services/sseService.ts`
  - `app/S14P11A201/smart-parking/src/services/vehiclePoseService.ts`
  - `app/S14P11A201/smart-parking/src/services/parkingCompleteService.ts`
  - `app/S14P11A201/smart-parking/src/services/exitLogService.ts`
  - `fe/S14P11A201/my-react-app/nginx.conf`

필요 정보
- SSE Base URL
- 인증 방식(Bearer 토큰 포함 여부)
- Reverse proxy 설정(`proxy_buffering off`, `proxy_read_timeout` 충분히 설정)

## 2) 인증/알림/결제

### Kakao Login
- 앱(모바일)에서 카카오 SDK 사용
- 참조 파일
  - `app/S14P11A201/smart-parking/app.json`

### Firebase FCM
- 앱 푸시 토큰 수집 + 서버 푸시 발송

필요 정보
- Firebase 프로젝트
- `google-services.json` (앱)
- 서버용 서비스계정 JSON (`firebase_service_account.json`)

- 모바일 결제 모듈
- 참조 파일

필요 정보

- 운영/테스트 CID 구분

## 4) 운영 전환 체크리스트
- 모든 하드코딩 키/비밀번호/IP를 환경변수로 치환.
- 카카오/Firebase/PortOne은 개발키와 운영키 분리.
- MQTT/DB/Redis 방화벽 및 TLS 정책 확인.
- SSE 장시간 연결을 위해 Proxy timeout/buffering 설정 확인(`fe` nginx 설정 반영).
