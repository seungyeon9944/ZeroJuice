# Zerojuice AI

## 1) 프로젝트 개요

**CCTV 영상 분석을 통한 실시간 주차 슬롯 점유 상태 감지 시스템**입니다.
고정된 CCTV 환경에서 사전에 정의된 주차 칸들의 영상 변화를 분석하여
각 슬롯의 점유 여부를 실시간으로 감지합니다.

딥러닝 기반 객체 탐지 대신 **ROI(Region of Interest) 기반 비전 알고리즘**을 사용하여
실시간성, 안정성, 유지보수성을 모두 확보한 솔루션입니다.

---

| 분야 | 기술 |
|------|------|

## 3) 주요 기능

### 1. **실시간 주차 슬롯 감지**
- 카메라 영상 프레임별 처리 및 분석
- ROI 영역 내 픽셀 변화량 기반 점유 판단
- 딥러닝 추론 없는 **극도로 빠른 처리** (거의 지연 없음)

### 2. **ROI 편집 도구 제공**
- `roi_editor.py`를 통한 주차 칸 위치 정의
- 마우스 클릭으로 각 슬롯의 4개 꼭짓점 설정
- JSON 형식으로 슬롯 정보 저장

### 3. **WebRTC 기반 실시간 데이터 전송**
- 감지된 점유 상태를 실시간으로 서버에 전송
- 비동기 처리로 안정적인 운영

### 4. **GPT 4o mini API 연동**
- GMS API를 통한 AI 분석 기능 확장 지원
- 설정 기반 확장 가능한 구조

### 5. **카메라 테스트 기능**
- `camera_test.py`로 카메라 연결 및 성능 확인
- 프레임 크기, 포맷 등 기본 설정 검증

## 5) 실행 방법

### 1️⃣ 사전 준비

**필수 라이브러리 설치:**
```bash
pip install opencv-python python-socketio python-dotenv numpy requests
```

### 2️⃣ 환경 설정

`.env` 파일 생성 및 설정:
```env
GMS_KEY=your_api_key_here
```

### 3️⃣ 주차 슬롯 ROI 정의 (첫 실행 시)

```bash
python roi_editor.py
```

**사용법:**
- 카메라 영상이 나타나면 주차 칸의 꼭짓점을 마우스로 클릭
- 각 주차 칸마다 4개의 점을 정의
- 완료 후 `slots.json` 자동 저장

### 4️⃣ 카메라 연결 테스트

```bash
python camera_test.py
```

### 5️⃣ 메인 시스템 실행

```bash
python parking_slot_detection.py
```

**동작:**
- CCTV 영상을 수신 (기본 카메라 ID: 0)
- 각 프레임마다 ROI 영역의 변화 분석
- 점유 상태 감지 및 Socket.IO를 통해 실시간 전송
- 필요시 GPT API를 활용한 추가 분석

### 주요 설정 항목

| **Socket.IO 서버** | `parking_slot_detection.py` | GMS 엔드포인트 설정 |

### 권장 환경

- **Python**: 3.8 이상
- **OS**: Windows, Linux, macOS
- **카메라**: USB 또는 내장 CCTV 연결 가능한 장치
- **네트워크**: Socket.IO 통신을 위한 인터넷 연결

### 주의사항

⚠️ **SSL 인증서**: 현재 코드에서 SSL 검증을 비활성화했습니다 (개발/테스트용).  
프로덕션 환경에서는 적절한 SSL 설정이 필요합니다.

⚠️ **API 키 보안**: `.env` 파일에 민감한 정보를 저장하므로  
버전 관리에서 제외(`gitignore`)해야 합니다.

## 7) 기술 선택 이유: YOLO 대신 ROI 기반 비전 알고리즘

### "문제를 풀기 위한 최소·최적의 도구를 선택했기 때문에 YOLO를 사용하지 않고 영상 변화 기반의 비전 알고리즘 사용"

YOLO 같은 딥러닝 객체 탐지는 범용성은 높지만,
이번 문제는 **카메라가 고정되어 있고 주차 칸 위치가 사전에 정의된 환경**이었기 때문에 **영상 변화 기반의 비전 알고리즘이 더 단순하고 안정적**이라고 판단합니다.

#### 1. 기술적 관점

YOLO는 객체의 bounding box를 검출하는 방식인데,
저희 문제는 '차가 어디에 있느냐'보다
**'해당 주차 칸이 점유되었는지'가 핵심**.
고정된 슬롯 ROI 내부의 변화량을 분석하는 것이
문제 정의에 더 직접적으로 부합합니다.

#### 2. 성능·운영 관점

⏱️ **실시간성**
- YOLO는 프레임마다 추론 비용이 발생
- 이 방식은 ROI 영역의 픽셀 연산만 수행하므로 **지연이 거의 없는 실시간 처리** 가능

🦾 **안정성**
- 딥러닝 모델은 데이터 분포가 달라지면 성능이 급격히 떨어질 수 있음
- 고정 환경에서는 규칙 기반 비전 알고리즘이 **환경 변화에 더 예측 가능하게 동작**

#### 3. 데이터 관점

YOLO를 사용하려면
카메라 각도·조명 조건에 맞는 차량 데이터셋과 라벨링이 필요합니다.
제한된 데이터 환경에서는
**학습 데이터에 의존하지 않는 방식이 더 신뢰성이 높다**고 판단했습니다.

#### 4. 유지보수 / 확장성 관점

주차 칸 위치가 바뀌거나 카메라가 추가될 경우,
딥러닝 모델을 재학습하는 대신
**ROI 재설정만으로 시스템을 빠르게 확장**할 수 있는 장점을 고려했습니다.

#### 5. 결론

YOLO는 범용 객체 탐지에 매우 강력한 모델이지만,
이번 문제는 고정된 CCTV 환경에서
사전에 정의된 주차 칸의 점유 여부를 판단하는 것이 목적입니다.
이 경우 객체 탐지보다
**ROI 기반 영상 변화 분석이 문제 정의에 더 직접적이고,
실시간성과 안정성, 데이터 의존성 측면에서도 유리**하므로
딥러닝을 사용하지 않고 컴퓨터 비전 기반 알고리즘을 선택했습니다.
